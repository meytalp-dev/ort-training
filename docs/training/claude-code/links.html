תרשימי זרימה

Step by Step – איך מתחילים, רואים ומפרסמים
▼
1
פתיחת תיקייה
בסייר הקבצים: צרו תיקייה חדשה. או בקלוד קוד: "צור תיקייה בשם my-project"
2
יצירת CLAUDE.md
בקלוד קוד: "צור CLAUDE.md - אני מורה ל[מקצוע], הפרויקט מיועד ל[מטרה], שפה: עברית"
3
מתחילים לעבוד!
כתבו מה אתם רוצים בעברית פשוטה. "תבנה לי...", "תצור לי...", "תעזור לי עם..."
אופציונלי: התקנת Plugins ו-Skills לשדרוג היכולות
1
מוצאים את הקובץ ב-Explorer
בסרגל השמאלי של VS Code רואים את כל הקבצים. מחפשים את index.html
2
קליק ימני → Reveal in File Explorer
לוחצים קליק ימני על הקובץ ובוחרים Reveal in File Explorer → נפתחת התיקייה בווינדוס
3
דאבל קליק → נפתח בדפדפן
דאבל קליק על קובץ ה-HTML → נפתח ב-Chrome/Edge. המשחק/האתר/המצגת עובדים!
1
שמירה (Commit)
בקלוד קוד: "תעשה commit לכל השינויים". קלוד שומר את העבודה עם הודעה מסודרת
2
העלאה (Push)
בקלוד קוד: "תעשה push ל-GitHub". העבודה עולה לענן ונגישה מכל מקום
3
אתר חי! (GitHub Pages)
בהגדרות הריפו ב-GitHub: הפעילו GitHub Pages. מקבלים קישור ציבורי: username.github.io/project
נלמד את זה לעומק במפגש 2 - בינתיים התוצר עובד מקומית על המחשב שלכם

טיפים – למתחילים

Tips & Best Practices
▼
דברו בעברית פשוטה
קלוד מבין עברית מצוין. "תבנה לי דף עם 3 כפתורים" עדיף על "תבנה דף יפה".
היו ספציפיים - אל תסמכו על ניחושים
ציינו קבצים, צבעים, מספרים. במקום "תוסיף שאלות" כתבו "תוסיף 5 שאלות אמריקאיות על נושא X".
Anthropic Docs
אל תקבלו את הגרסה הראשונה
תמיד אפשר לשפר: "שנה צבע", "הגדל פונט", "הוסף אנימציה", "תקצר טקסט". קלוד משתפר בכל בקשה.
תנו לקלוד לראיין אתכם
במקום לנסח הכל בעצמכם, כתבו: "אני רוצה לבנות [תיאור]. שאל אותי שאלות על מה שאני צריך".
Reddit Community
Explore → Plan → Code
שלושת השלבים: חקרו את הקוד, תכננו את השינוי, רק אז בנו. זה הסדר שאנתרופיק ממליצים.
Anthropic Best Practices
תכננו לפני שבונים (Plan Mode)
Shift+Tab → Plan Mode. קלוד מציג תכנית בלי לבצע. רק אחרי שאתם מרוצים - נותנים לו לעבוד. חוסך שעות של תיקונים.
בנו בשלבים קטנים
אל תבקשו "תבנה מערכת שלמה". בנו חלק אחד, בדקו, שמרו, והמשיכו הלאה.
SFEIR Institute
שיחה חדשה לנושא חדש
ככל שהשיחה ארוכה יותר, הביצועים יורדים. /clear בין משימות, או פתחו שיחה חדשה כשעוברים נושא.
שמרו תמיד עם Commit
אחרי כל שינוי שעובד, בקשו: "תעשה commit". אם משהו נשבר אח'כ - חוזרים לגרסה התקינה בשניות.
אל תתנו משימה ענקית
משימות גדולות מדי מובילות לתוצאות חלקיות. פרקו לחלקים קטנים, כל חלק בנפרד, ובדקו אחרי כל שלב.
SFEIR Institute
השתמשו ב-/config לבחירת סגנון
הקלידו /config ובחרו סגנון תשובות: Explanatory (מפורט), Concise (קצר), Technical (מקצועי).

עצרו מוקדם - אל תתנו לטעויות להצטבר
אם קלוד עשה משהו לא נכון, לחצו Escape מיד. אחרי 2 תיקונים כושלים, עשו /clear והתחילו עם הנחיה טובה יותר.
הרשאות - לא להיבהל
כשקופץ "Allow?" - קלוד מבקש אישור לפעולה. קראו מה הוא רוצה ולחצו Allow. אפשר גם "Allow for this project" כדי שלא ישאל שוב.
אם כותב באנגלית
פשוט כתבו: "תכתוב הכל בעברית". או הוסיפו ל-CLAUDE.md: "שפה: עברית" - ולא תצטרכו לבקש שוב.
קראו לפני שמאשרים
12% מפקודות מעורפלות מכילות פעולות שעלולות למחוק קבצים. תמיד קראו מה קלוד רוצה לעשות לפני שלוחצים Allow.
SFEIR Institute

טיפים – למתקדמים

Skills, Automation & Performance
▼
השתמשו ב-Subagents לחקירות
כשקלוד חוקר את הקוד הוא ממלא את חלון ההקשר. Subagents עובדים בחלון נפרד ומחזירים רק סיכום.
צרו מסמך Handoff לפני שמתחילים מחדש
כתבו: "תעד את התוכנית ב-HANDOFF.md. מה ניסית, מה הצליח ומה לא, כדי שהסשן הבא ימשיך".
/compact עם הנחיה
"/compact התמקד בשינויי העיצוב" - מכוון מה לשמור כשדוחסים את השיחה.
Anthropic Docs
CLAUDE.md קצר = ביצועים טובים
אם CLAUDE.md ארוך מדי (מעל 300 שורות), קלוד מתחיל להתעלם מהוראות. שמרו קצר וממוקד.
Reddit Community
צרו Skills - פקודות מותאמות אישית
Skills הם תבניות עבודה חוזרות. צרו תיקיית .claude/skills/ עם קובץ SKILL.md.
הגדירו Hooks - פעולות אוטומטיות
Hooks רצים אוטומטית בנקודות מסוימות. למשל: "כתוב hook שמריץ בדיקה אחרי כל עריכת קובץ".
הפרידו כללים קבועים מהקשר זמני
שמרו כללים קבועים בתיקיית .claude/rules/. את CLAUDE.md השאירו לדברים שמשתנים.
awesome-claude-code
ריפו GitHub עם אוסף קהילתי של סקילס, hooks ו-plugins מוכנים לשימוש.
GitHub Community
השתמשו במודלים שונים למשימות שונות
Opus למשימות מורכבות. Haiku לעבודות פשוטות - חוסך טוקנים.
עבדו עם כמה מופעים במקביל
פתחו כמה טאבים, בכל אחד claude בתיקייה שונה. הגבילו ל-3-4.
דברו לקלוד בקול
כלי תמלול קולי כמו SuperWhisper מאיצים את העבודה. קלוד מבין גם תמלול לא מושלם.
