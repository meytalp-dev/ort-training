תרשים זרימה -- בניית סקיל
לחצו על כל שלב לפרטים נוספים
1
הגדרת המטרה
מה הסקיל צריך לעשות?
איזו בעיה הסקיל פותר? מי משתמש בו? מה התוצר הסופי שצריך לצאת? ככל שהמטרה ברורה יותר -- הסקיל יעבוד טוב יותר.
2
בחירת שם
שם ברור, קצר, באנגלית
שם בפורמט kebab-case (מילים מופרדות במקף). למשל: create-presentation, build-quiz, lesson-builder. השם צריך לתאר את הפעולה.
3
כתיבת תיאור
משפט שגורם לקלוד להפעיל את הסקיל
התיאור צריך להיות ברור ותמציתי. הוא עוזר ל-Claude Code לזהות מתי להציע את הסקיל -- גם בלי שנקליד את השם שלו ישירות.
4
הגדרת טריגרים
מתי הסקיל מופעל?
באילו מילות מפתח הסקיל מופעל? מתי כן ומתי לא? למשל: "כשהמשתמש מבקש ליצור מצגת" -- כן. "כשהמשתמש שואל שאלה כללית" -- לא.
5
כתיבת הפרומפט
הלב של הסקיל
ההוראות המפורטות -- מה לעשות, בסדר, עם מגבלות. ככל שהפרומפט ברור ומפורט יותר, התוצר שיוצא טוב יותר. חלקו לסעיפים ושלבים.
6
הוספת דוגמאות
קלט צפוי ופלט רצוי
הראו ל-Claude Code דוגמאות של מה אתם מצפים לקבל. דוגמה אחת טובה שווה יותר מעשרה משפטי הסבר.
7
יצירת תבניות
קבצי template לשימוש חוזר
קבצי template שהסקיל יכול להשתמש בהם כבסיס. למשל: תבנית HTML למצגת, תבנית מבחן, מבנה שיעור. שומרים אותם באותה תיקייה.
8
בדיקה ושיפור
מפעילים, בודקים, משפרים, חוזרים
הפעילו את הסקיל עם קלטים שונים. בדקו את התוצר. מה טוב? מה חסר? שפרו את הפרומפט וחזרו שוב. זה תהליך איטרטיבי.

מ-Claude Code לסוכן אוטונומי
מה זה סוכן?
Claude Code עם אישיות, תפקיד, ויכולות מותאמות
סוכן = Claude Code עם הנחיות מותאמות
הגדרה דרך קובץ CLAUDE.md שמספר לקלוד מי הוא, מה התפקיד שלו, ומה הכלים שלו
ההבדל המרכזי
סקיל = פקודה ספציפית  |  סוכן = אישיות שלמה
דוגמאות לסוכנים
סוכן שבודק איכות  |  סוכן שכותב תוכן  |  סוכן שבונה שיעורים  |  סוכן שמנהל מיילים
חשבו על זה ככה: סקיל זה כמו מתכון. סוכן זה כמו שף -- הוא יודע הרבה מתכונים, מכיר את הטעם שלכם, ויודע מה לבשל מתי.
תרשים זרימה -- בניית סוכן
לחצו על כל שלב לפרטים נוספים
1
הגדרת תפקיד
מי הסוכן? מה המומחיות שלו?
הגדירו בדיוק מה הסוכן עושה. "סוכן QA שבודק מצגות HTML" עדיף על "סוכן שעוזר". ככל שהתפקיד ממוקד -- הסוכן עובד טוב יותר.
2
בחירת כלים
אילו כלים הסוכן צריך?
קבצים, אינטרנט, API, Gmail, Sheets, Calendar -- בחרו את הכלים שרלוונטיים לתפקיד. אל תתנו לסוכן כלים שהוא לא צריך.
3
כתיבת הנחיות מערכת
ה-CLAUDE.md שמגדיר את ההתנהגות
ה-CLAUDE.md הוא "המוח" של הסוכן. כתבו שם את הכללים, הסגנון, ההעדפות, והמגבלות. זה מה שהופך את Claude Code לסוכן שלכם.
4
הגדרת קלט/פלט
מה הסוכן מקבל? מה הוא מחזיר?
הגדירו מה הסוכן מצפה לקבל (נושא? קובץ? רשימה?) ומה הוא צריך להחזיר (קובץ HTML? טבלה? מייל?). ככל שזה ברור -- פחות הפתעות.
5
הגדרת גבולות
מה הסוכן לא אמור לעשות
הגדירו מה אסור -- "אל תמחק קבצים", "אל תשנה קבצי CSS קיימים", "אל תוסיף אימוג'ים". הגבולות מונעים טעויות ושומרים על שליטה.
6
בדיקה
תרחישים שונים, קלטים לא צפויים
בדקו עם תרחישים שונים: קלט רגיל, קלט חסר, קלט לא צפוי. ראו איך הסוכן מגיב ואם הוא נשאר בתפקיד.
7
שיפור
למידה מטעויות, עדכון ההנחיות
כל טעות היא הזדמנות לשפר. הוסיפו כלל ל-CLAUDE.md, עדכנו דוגמה, חדדו גבול. הסוכן ילמד ויהפוך טוב יותר עם הזמן.
