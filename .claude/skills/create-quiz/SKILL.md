# Skill: צור שאלון – /create-quiz

כאשר המשתמש מבקש **ליצור שאלון** על נושא כלשהו – יש להפעיל את הסקיל הזה.

---

# קלט חובה

המשתמש **חייב** לספק ארבעה פרטים:

1. **נושא** – על מה השאלון
2. **כיתה** – ט / י / יא / יב
3. **מקצוע** – עברית / מתמטיקה / היסטוריה / אנגלית / מדעים / חשבונאות / אחר
4. **מייל מורה** – כתובת המייל של המורה שייצר את השאלון

אם חסר אחד מהם – **שאל את המשתמש לפני שתמשיך**.

**הרמה לא נדרשת** – השאלון דיפרנציאלי ומזהה את רמת התלמיד אוטומטית מהשאלה המאבחנת.

**מייל המורה** חשוב כי:
- כל מורה מקבל tab נפרד בגיליון עם השם שלו
- התוצאות מסומנות לפי מורה כדי שכל מורה יראה רק את הציונים שלו
- המייל מוטמע בתוך קובץ ה-HTML (לא צריך להקליד בכל פעם)

המקצוע קובע את **סוג השאלות**:
- **עברית/דקדוק**: זיהוי חלקי דיבר, ניתוח משפטים, הבנת הנקרא
- **מתמטיקה**: חישוב, פתרון בעיות, זיהוי נוסחאות
- **היסטוריה**: תאריכים, אירועים, קשרי סיבה-תוצאה
- **אנגלית**: אוצר מילים, דקדוק, הבנת הנקרא
- **מדעים**: מושגים, תהליכים, ניסויים
- **חשבונאות**: פקודות יומן, חישובים, מושגים

דוגמה לשימוש:
```
/create-quiz נושא: שם המספר | כיתה: י | מקצוע: עברית | מורה: dana@bethaarava.ort.org.il
```

קלט אופציונלי נוסף:
- קובץ מצורף (PDF / DOCX / PPTX / טקסט) – הכלי ילמד את החומר וייצר שאלות ממנו
- מספר שאלות מותאם (ברירת מחדל: 10)
- רמה ספציפית (אם המורה רוצה לקבע רמה במקום אבחון אוטומטי)

# מבנה השאלון

## 1. מסך פתיחה

- שם השאלון
- שדה להזנת שם התלמיד (חובה)
- שדה להזנת כיתה (חובה)
- כפתור "התחל"

## 2. שאלות מאבחנות (שאלות 1–2)

שתי השאלות הראשונות הן **שאלות מאבחנות** ברמה בינונית.

מטרתן: לזהות את רמת התלמיד **מאחורי הקלעים** ולשבץ אותו לאחת משלוש רמות.

### חשוב מאוד – דיפרנציאליות סמויה

- **התלמיד לא יודע שהוא מאובחן**
- **אין שום סימון של "רמה" בממשק**
- **השאלון נראה זהה לכל התלמידים**
- הכל קורה בקוד ה-JavaScript ברקע

### לוגיקת האבחון (2 שאלות → 3 רמות)

```
0 תשובות נכונות מתוך 2 → רמה בסיסית → שאלות פשוטות
1 תשובה נכונה מתוך 2  → רמה בינונית → שאלות ברמה סטנדרטית
2 תשובות נכונות מתוך 2 → רמה מתקדמת → שאלות מאתגרות
```

### יישום טכני

יש להכין **שלוש ערכות שאלות** (אחרי 2 השאלות המאבחנות):

- **ערכה א' (בסיסית):** שאלות של זיהוי, התאמה, שינון, הבנה ישירה
- **ערכה ב' (בינונית):** שאלות של הבנה, יישום, קישור בין מושגים
- **ערכה ג' (מתקדמת):** שאלות של ניתוח, השוואה, חשיבה ביקורתית

שלוש הערכות על **אותו נושא** – רק ברמת קושי שונה.

שלוש הערכות באותו אורך (אותו מספר שאלות).

**התלמיד לא רואה שום הבדל** – אותו עיצוב, אותו מספר שאלות, אותו ציון מ-100.

## 3. שאלות (3–10)

- שאלות אמריקאיות (4 אפשרויות)
- שאלה אחת בכל מסך
- אחרי בחירת תשובה: משוב מיידי (נכון ✓ / שגוי ✗)
- הסבר קצר לתשובה הנכונה
- כפתור "שאלה הבאה"

### חשוב מאוד – ערבוב תשובות

**התשובה הנכונה חייבת להיות במיקום אקראי שונה בכל שאלה!**

- בקוד, הגדר את השאלות עם `correctIndex` שמצביע על מיקום התשובה הנכונה במערך
- **פזר את `correctIndex` באופן שווה** בין 0, 1, 2, 3 לאורך כל השאלות
- לדוגמה: אם יש 8 שאלות, כל מיקום (א/ב/ג/ד) צריך להופיע כ-2 פעמים כתשובה נכונה
- **אסור** שהתשובה הנכונה תהיה תמיד באותו מקום (למשל תמיד אפשרות א')
- ודא שהמסיחים הגיוניים ולא ניתנים לפסילה מיידית

## 4. מסך סיום

- ציון סופי (מספר ואחוז)
- משוב כללי:
  - 90-100: מצוין!
  - 70-89: כל הכבוד!
  - 50-69: אפשר לשפר
  - 0-49: כדאי לחזור על החומר
- כפתור "שלח תוצאות"
- לאחר השליחה: הודעת אישור

### נתונים לשליחה

// שליחה באמצעות GET (script tag) – עובד תמיד בלי בעיות CORS

  action: 'submit',
  quizName: "שם השאלון",
  studentName: "שם התלמיד",
  classroom: "כיתה",
  score: 85,
  totalQuestions: 10,
  correctAnswers: 8,

});
const script = document.createElement('script');
script.src = GOOGLE_SHEETS_URL + '?' + params.toString();
document.body.appendChild(script);

# מבנה מאגר השאלונים

הקובץ `docs/quizzes/index.html` הוא דף שמרכז את כל השאלונים שנוצרו.

הדף כולל:
- כותרת "מאגר שאלונים – אורט בית הערבה"
- **סינון לפי מקצוע** (כפתורים: הכל / עברית / מתמטיקה / היסטוריה / וכו')
- **חיפוש חופשי** (לפי שם שאלון או נושא)
- רשימת כרטיסים – כל שאלון ככרטיס עם:
  - שם השאלון
  - מקצוע (תגית צבעונית)
  - כיתה
  - שם מורה
  - כפתור "פתח שאלון" (קישור)

עיצוב: אותו סגנון אורט (צבעים, פונטים, CSS variables).

כל פעם שנוצר שאלון חדש, יש להוסיף כרטיס חדש למאגר.
