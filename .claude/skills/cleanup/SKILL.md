# סקיל ניקוי ותחזוקה

סקיל לניקוי שוטף של הפרויקט — קבצי temp, זיכרון מיושן, וקבצים לא מנוהלים בריפו.

## שלב 1 — ניקוי קבצי זבל

סרוק את שורש הפרויקט ומחק:
- `temp-*` תיקיות (frames, reel, וכו')
- `*.bak` קבצים
- `WhatsApp Image*` קבצים
- `צילום מסך*` קבצים
- קבצי `0.png`, `test-*` חד-פעמיים
- תיקיות ריקות

**הצג** רשימת מה נמחק + כמה MB שוחררו.

## שלב 2 — בדיקת זיכרון

קרא את `MEMORY.md` מהתיקייה:
```
C:\Users\USER\.claude\projects\c--Users-USER-Downloads-ort-presentation-builder\memory\
```

לכל קובץ זיכרון מסוג `project`:
1. קרא את הקובץ
2. בדוק תאריך עדכון אחרון (מתוך התוכן או mtime)
3. אם הפרויקט מסומן "בתהליך"/"בתכנון"/"ממתין" ולא עודכן 30+ יום — סמן כ**חשוד למחיקה**

**הצג** טבלה:
| קובץ | סטטוס | עדכון אחרון | המלצה |
|-------|--------|-------------|--------|

**אל תמחק זיכרון אוטומטית** — רק הצג המלצות ותחכה לאישור.

## שלב 3 — קבצים לא מנוהלים בריפו

הרץ `git status` (בלי `-uall`) וזהה:
- קבצים untracked גדולים (>1MB)
- קבצים untracked בשורש (לא בתיקיות)
- תיקיות untracked שנראות זמניות

**הצג** טבלה:
| קובץ | גודל | המלצה (gitignore / commit / delete) |
