# Skill: הדרכה מלאה – /training

סקיל-על שמפעיל את כל מערכת ההדרכות. כאשר המשתמש מבקש **ליצור הדרכה על כלי** (Canva, Gemini, Padlet, וכו') – הסקיל הזה מתאם את כל התהליך.

---

## קלט

המשתמש מספק אחד או יותר מאלה:

- שם כלי (לדוגמה: "תעשה לי הדרכה על Canva AI")
- הקלטת מסך (MP4) של שימוש בכלי
- קובץ תוכן (PPTX / PDF / DOCX / טקסט)
- נושא ספציפי בתוך כלי (לדוגמה: "מדיה קסם בקנבה")

---

## תהליך עבודה – Pipeline

### שלב 1: חקר תוכן (`/ai-expert`)
- חקר עמוק של הכלי
- מציאת ערך פדגוגי וחיסכון זמן
- כתיבת תוכן מותאם לקהל היעד

### שלב 2: חילוץ תמונות (`/extract-frames`)
- אם יש הקלטת מסך – חילוץ פריימים מפתח
- אם אין הקלטה – דילוג (שימוש בתמונות מהרשת)

### שלב 3: עיבוד תמונות (`/image-process`)
- הוספת annotations (חצים, הדגשות)
- זום לאזורים חשובים
- יצירת GIF מצעדים (אופציונלי)

### שלב 4: בניית תוצרים
הסקיל מפעיל את הסקילים הבאים לפי הצורך:

| סקיל | תוצר | חובה? |
|-------|-------|-------|
| `/create-presentation` | מצגת HTML אינטראקטיבית | ✔ חובה |
| `/create-reels` | סרטון רילס 60 שניות | ✔ חובה |
| `/infographic` | אינפוגרפיקה לשיתוף | אופציונלי |
| `/practice-sim` | סימולציית תרגול אינטראקטיבית | אופציונלי |

### שלב 5: פרסום (`/publish`)
- שמירה ב-`docs/`
- git add + commit + push
- עדכון דף הבית של האתר
- הצגת קישורים חיים

## פלט סופי

לכל הדרכה נוצרת תיקייה:

```
docs/training/{tool-name}/
├── index.html          ← מצגת ראשית
├── reel-9x16.mp4       ← רילס לטיקטוק/אינסטגרם
├── reel-1x1.mp4        ← רילס מרובע
├── infographic.html    ← אינפוגרפיקה (אופציונלי)
└── practice.html       ← סימולציה (אופציונלי)
```

## שאלות לפני הפעלה

לפני שמתחילים, לשאול את המשתמש:

1. על איזה כלי ההדרכה?
2. יש הקלטת מסך? (אם כן – לבקש את הקובץ)
3. אילו תוצרים רוצים? (מצגת / רילס / אינפוגרפיקה / סימולציה)
4. באיזה אתר לפרסם? (Training / Lovable / שניהם)
